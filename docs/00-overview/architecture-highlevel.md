# システムアーキテクチャ概要

## システム構成図

```
[ユーザー]
    ↓
[Webブラウザ（React + TanStack Start）]
    ↓ HTTP/HTTPS
[APIサーバー（Go + Gin）]
    ↓
[データアクセス層（Ent）]
    ↓
[PostgreSQL データベース]
```

## レイヤー構成

### フロントエンド層

- **技術スタック**: React + TanStack Start
- **責務**:
  - ユーザーインターフェースの提供
  - ユーザー入力の検証
  - API サーバーとの通信
  - 状態管理

### API サーバー層

- **技術スタック**: Go + Gin
- **責務**:
  - REST API エンドポイントの提供
  - ビジネスロジックの実装
  - 認証・認可の処理
  - データの検証とバリデーション

### データアクセス層

- **技術スタック**: Ent（ORM フレームワーク）
- **責務**:
  - データベースとのやり取り
  - クエリの生成と最適化
  - トランザクション管理

### データベース層

- **技術スタック**: PostgreSQL
- **責務**:
  - データの永続化
  - データ整合性の保証
  - クエリの実行

## 通信プロトコル

- フロントエンド ⇔ API サーバー: HTTP/HTTPS（REST API）
- API サーバー ⇔ データベース: PostgreSQL プロトコル

## セキュリティ

- 本番環境では HTTPS を使用
- API リクエストには適切な認証トークンを付与
- ユーザーデータは完全に隔離され、他ユーザーのデータへのアクセスを防止
